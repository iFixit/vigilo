# Vigilo

![Version: 0.1.0](https://img.shields.io/badge/Version-0.1.0-informational?style=flat-square) ![Type: application](https://img.shields.io/badge/Type-application-informational?style=flat-square) ![AppVersion: 1.0.0](https://img.shields.io/badge/AppVersion-1.0.0-informational?style=flat-square)

A Helm chart for Kubernetes. Vigilo is a monitoring tool that automates Lighthouse audits and sends key metrics to Datadog. Designed to keep a vigilant eye on your web application's performance and accessibility. This chart will deploy the Vigilo application as a CronJob.

## Values

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| datadogSecret.apiKey | string | `nil` | A `base64` encoded string of your Datadog API Key. This will be set as an environment secret for the container. |
| datadogSecret.appKey | string | `nil` | A `base64` encoded string of your Datadog APP Key. This will be set as an environment secret for the container. |
| datadogSecret.secretName | string | `vigilo-datadog-secret` | Name of the secret where to retrieve the Datadog keys from.  |
| existingDatadogSecret | string | `""` | If set, then Vigilo will use the keys defined from the existing secret instead of creating one. |
| lhConfig | string | *`rawJS`** | A raw JS string to configure lighthouse. Visit https://github.com/GoogleChrome/lighthouse/blob/main/docs/configuration.md#more-examples to learn more about the configuration options for Lighthouse. |
| urls[] | object | `Test: ["https://www.wikipedia.org"]` | A `key:value` pair of urls to audit. Where the `key` is the page type and the `value` is an array or urls. This will then be converted to `JSON` file for the Vigilo pod. |
